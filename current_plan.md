# Current plan

## Goal

Establish `groundtruth/corpus/latex/` as the canonical, version-controlled corpus root, with one folder per document. Replace ad-hoc generated `.tex` files with first-class source-of-truth fixtures. Compile every fixture deterministically with `lualatex` producing a tagged PDF (PDF/UA-2 via `\DocumentMetadata`). Certify every compiled PDF against a fixed set of blocking checks; only certified documents count as ground truth. All downstream pipeline scripts must read the corpus from this new root.

## Scope and non-goals

In scope: corpus migration, compilation tool, certification tool, path updates in existing pipeline scripts, README update.

Out of scope: changes to consensus, semantic linker, builder, or comparator logic. Semantic-linking parity work resumes in the next plan, on top of the certified corpus.

Single-file justification:

- `compile_corpus.py` and `certify_corpus.py` are independent tools with their own CLI and lifecycle (compile is idempotent and hash-gated; certify is read-only and exit-coded). They will be invoked separately by humans, by CI, and by other scripts. Splitting them is not premature; merging them would conflate read-only verification with write-side compilation.
- `tests/test_corpus_tooling.py` is a single test file covering both tools; no separate test files justified at this stage.
- All other concepts (path updates, doc updates) live in pre-existing files.

## Per-document layout (under `groundtruth/corpus/latex/<doc_id>/`)

- `<doc_id>.tex`             source of truth, version-controlled
- `<doc_id>.bib`             optional, version-controlled if present
- `assets/`                  optional, for included figures
- `meta.toml`                version-controlled metadata; see schema below
- `<doc_id>.pdf`             generated by `compile_corpus.py`, gitignored
- `build.log`                generated by `compile_corpus.py`, gitignored
- `certification.json`       generated by `certify_corpus.py`, gitignored

`meta.toml` schema:

    document_id = "<doc_id>"
    documentclass = "article"            # or report, book, etc.
    expected_features = ["equations", "figures", "tables", "footnotes", "references", "bibliography"]
    expected_counts = { sections = 3, figures = 2, tables = 1, equations = 5, footnotes = 2 }
    notes = "free text"

Engine and PDF standard are fixed for the whole corpus and not declared per fixture: `lualatex` with `\DocumentMetadata{testphase=phase-III, pdfstandard=ua-2}` required at the top of every `.tex`. Certification enforces this.

## Tasks

### T1 — Migrate corpus to canonical layout

Extract every `.tex` currently embedded in `create_latex_examples.sh` (and any other generator) into versioned files under `groundtruth/corpus/latex/<doc_id>/<doc_id>.tex`. For each document:

- ensure `\DocumentMetadata{testphase=phase-III, pdfstandard=ua-2}` is the first non-comment line, before `\documentclass`,
- ensure `\usepackage{hyperref}` is present (or implied by the metadata mode),
- write a `meta.toml` with declared counts derived by inspection of the `.tex`,
- if a `.bib` or `assets/` are referenced, place them at the documented locations.

Files: `groundtruth/corpus/latex/**`.

### T2 — `scripts/compile_corpus.py`

Walk the corpus, compile each fixture with `lualatex` (twice, plus `biber` if a `.bib` is present and the `.tex` references citations). Capture `build.log`. Idempotent: if `sha256(<doc_id>.tex) + sha256(<doc_id>.bib) + sha256(assets/*)` matches a stored hash in the `build.log` header, skip. CLI:

    python scripts/compile_corpus.py --corpus-root groundtruth/corpus/latex [--doc <id>] [--force]

Exit 0 on full success. Exit non-zero with a per-document summary on any compilation failure (parsed from the `.log`: `! ` lines, `Reference … undefined`, `Citation … undefined`).

Files: `scripts/compile_corpus.py`.

### T3 — `scripts/certify_corpus.py`

Walk the corpus, run blocking checks per document, write `certification.json` per document and `corpus_report.json` at the corpus root. CLI:

    python scripts/certify_corpus.py --corpus-root groundtruth/corpus/latex [--doc <id>]

Blocking checks (any failure → document fails certification):

1. `<doc_id>.pdf` exists and is newer than `<doc_id>.tex`.
2. `build.log` shows zero `! ` errors, zero `Reference … undefined`, zero `Citation … undefined`.
3. `pikepdf` opens the PDF and finds `/Catalog/Names/Dests` non-empty.
4. `/Catalog/StructTreeRoot` is present and non-empty (PDF is tagged).
5. Count of `\label` in `.tex` equals count of named destinations in PDF (modulo hyperref prefixes).
6. Each `\ref / \eqref / \autoref` in the `.tex` resolves to a `/Link` annotation with a destination found in `/Names/Dests`.
7. Each `\cite` resolves to a `/Link` to a bibliography entry in the rendered PDF.
8. `pymupdf` extracts non-empty selectable text from every page.
9. PDF outline (bookmarks) covers all `\section` declarations in the `.tex`.
10. `meta.toml` `expected_counts` match observed counts in the PDF (tolerance 0).
11. `\DocumentMetadata` is present in the `.tex` with the declared phase and standard.

Warning checks (recorded but non-blocking):

- `Overfull`/`Underfull` boxes in `build.log`.
- Fonts not embedded.
- StructTree depth shallower than declared section depth (suggests partial tagging).

`certification.json` per document includes per-check status and the failing reason for any block. `corpus_report.json` aggregates and exits 0 only if every document passes.

Files: `scripts/certify_corpus.py`.

### T4 — Update pipeline scripts to read from the new corpus root

Add a `--corpus-root` flag (default `groundtruth/corpus/latex`) to:

- `latex_to_pre_docling_groundtruth.py`
- `generate_latex_docling_groundtruth.py`
- `validate_latex_docling_groundtruth.py`
- `compare_pre_docling_groundtruth.py`

In `latex_to_pre_docling_groundtruth.py`: change the per-document path from `<doc>/input/<doc>.tex` to `<doc_id>/<doc_id>.tex`. Same for the PDF.

Files: as listed.

### T5 — Update README harness and `.gitignore`

Update `README_latex_docling_groundtruth.md` to describe the canonical corpus layout, the compile→certify→derive flow, and remove references to `create_latex_examples.sh`. Files: `README_latex_docling_groundtruth.md`, `.gitignore`.

## Tests

Tests are automated by default. A test re-tagged `human` in `run_log.md` after a verifiable environmental failure is reported but does not block `ready_for_review`.

### A1 — Unit: compile_corpus hash gating
- command: `pytest -q tests/test_corpus_tooling.py -k "compile and hash"`
- pass: with `lualatex` invocation mocked, the script computes input hashes correctly, skips when stored hash matches, recompiles when any input changes or `--force` is passed.

### A2 — Unit: compile_corpus log parsing
- command: `pytest -q tests/test_corpus_tooling.py -k "compile and log"`
- pass: detects `! ` errors, `Reference … undefined`, `Citation … undefined`, and surfaces them with document id and line number.

### A3 — Unit: certify_corpus per-check
- command: `pytest -q tests/test_corpus_tooling.py -k certify`
- pass: each blocking check is exercised on a synthetic minimal PDF + companion `.tex`. Includes one passing case and one failing case per check.

### A4 — Integration: compile + certify on a single fixture
- command: `pytest -q tests/test_corpus_tooling.py -k integration`
- pass: a tiny in-tree fixture (one section, one labeled equation, one figure, one ref, one cite) compiles via real `lualatex` and certifies with all blocking checks green. Re-tagged human in `run_log.md` if `lualatex` is not on `PATH` in the agent sandbox.

### A5 — Pipeline smoke: scripts read from the new corpus root
- command: `pytest -q tests/test_corpus_tooling.py -k pipeline_paths`
- pass: each updated pipeline script accepts `--corpus-root`, defaults to the canonical path, and fails with a clear message if the path does not exist.

### H1 — Full corpus: compile + certify
- tag: human (likely; depends on whether the agent sandbox has texlive 2023+ with lualatex and biber).
- command: `python scripts/compile_corpus.py --corpus-root groundtruth/corpus/latex && python scripts/certify_corpus.py --corpus-root groundtruth/corpus/latex`
- pass: every document under the corpus root produces a certified PDF; `corpus_report.json` exits 0.

### H2 — Visual inspection of one tagged PDF
- tag: human.
- command: open one PDF in a viewer that exposes the structure tree (Adobe Reader → View → Show/Hide → Navigation Panes → Tags; or `pdfinfo -struct-text` on Poppler ≥ 22).
- pass: the structure tree is visible and contains expected `<Sect>`, `<Figure>`, `<Caption>`, `<Formula>`, `<Table>`, `<Note>` nodes for the document inspected.
